## Rspressについて

公式ドキュメントにはこう書いてる。

> ## Rspressを選ぶ理由
>
> Rspress は次のコア機能を中心に設計されています。
>
> - ビルドパフォーマンス。十分に速い起動速度を確保することで、優れた開発エクスペリエンスが実現します。
> - AIネイティブ。技術文書は人間の読者に役立つだけでなく、SSG-MDを通じてAIによる理解と活用も向上します。
> - MDX サポート。MDX を使用すると、ドキュメント フラグメントを簡単に再利用し、ドキュメント内でカスタム React コンポーネントをレンダリングできます。
> - ドキュメントサイトの基本機能。国際化、複数バージョンのサポート、全文検索、コンポーネントライブラリのドキュメントなどが含まれます。
> - 拡張性。組み込みのプラグイン システムを提供し、プラグイン API を通じて Rspress の拡張をサポートします。

個人的には基本機能が充実していて好みだった。
セットアップしただけで[何も設定しなくてもヘッダーやサイドバーが結構リッチ](https://rspress.rs/guide/start/introduction#automatic-layout-generation)で、見た目の細かい調整を面倒がる自分的には好み。

ホットリロードは初期設定ではなさそうだが、追加できるのだろうか。

### 初期設定の場所

自動で設定されているヘッダーやサイドバーは`rspress.config.ts`の`themeConfig`で設定されているようだった。
デフォルトで結構モリモリ設定されていて、都度上書いていく感じのようだ

今はカスタムする気が起きないのでカスタムしない。
